# MID-60-tools

Локальний інструмент для підрахунку й інтерпретації **Multidimensional Inventory of Dissociation – 60-item version (MID-60)**. Парний до [ampd-tools](https://github.com/BenBakster/ampd-tools) — звичайний HTML, без bundler'а; усі обчислення відбуваються у браузері клієнта.

## Як запустити

`index.html` через `file://` не працює — JSON-дані завантажуються через `fetch()`, який блокується CORS у локальному файлі. Потрібен будь-який локальний static server:

```bash
cd mid60-tools
python3 -m http.server 8080
# Далі — http://localhost:8080
```

## Що всередині

- 60-пунктовий опитувальник для самозаповнення (типово — українська, літературний клінічний переклад; англійська — валідована; перемикач у верхньому правому куті)
- Підрахунок: загальний середній × 10, 12 середніх по підшкалах, рівні тяжкості
- Перцентилі: загальна популяція (Φ(z), μ=12.94 σ=13.32) + клінічна вибірка (μ=56.8 σ=18.8)
- 16-категорійна Probable Profile Classification (Kate 2025)
- Пункти безпеки (запитання 22 / 44 / 58) — авто-сповіщення про самоушкодження / суїцидальні думки
- Композитне ранжування `0.5 × ((score-cutoff)/cutoff) + 0.5 × (items_at_threshold/N)` для відбору найбільш виражених підшкал
- Дослівні інтерпретативні абзаци з OSF Technical Paper
- Звіт для друку / збереження в PDF

## PHI та приватність

- Усі дані лишаються у браузері — жодних зовнішніх запитів.
- Чернетка опитувальника зберігається у `sessionStorage` поточної вкладки на 1 годину.
- У `localStorage` тримається тільки вибір мови інтерфейсу.
- Для довгострокового запису — друк звіту у PDF (Ctrl+P → «Зберегти як PDF»).
- `.gitignore` виключає `responses/`, `*.responses.json`, `*.session.json`.

## Клінічне попередження

Скринінг, не діагноз. Застосовувати тільки разом із клінічним інтерв'ю і (за показаннями) SCID-D / DDIS / TADS-I.

При спрацьованому пункті безпеки (22 / 44 / 58):
1. Пряма оцінка суїцидальних думок (за C-SSRS або клінічним протоколом установи).
2. Інформована згода на контакт із близькими.
3. Ескалація залежно від ризику.

## Інтерпретація звіту

Стислий порядок читання:

1. **Пункти безпеки** — насамперед. Якщо спрацювали 22 / 44 / 58 — далі за протоколом ризику (C-SSRS).
2. **Загальний бал → ступінь тяжкості** (Немає / Мінімальна / Легка / Помірна / Висока / Тяжка / Надзвичайна).
3. **Ймовірний профіль** — 16 категорій, hierarchical від найтяжчого. Це **гіпотеза для верифікації**, не діагноз.
4. **Топ-підшкали** за композитним ранжуванням (tightness + breadth) + інтерпретативні абзаци.

Повний клінічний гід (опис 16 профілів, 12 підшкал, маршрутизація до SCID-D / DDIS / TADS-I, DD-confounds) — **[docs/INTERPRETATION.md](docs/INTERPRETATION.md)**.

## Структура

```
mid60-tools/
  index.html              # опитувальник + рендер звіту
  mid60-engine.js         # чистий алгоритм підрахунку (без DOM)
  data/
    items.json            # 60 пунктів: текст / підшкала / пункт безпеки
    subscales.json        # 12 підшкал: пункти / межа / інтерпретативний текст
    classification.json   # 16 діагностичних профілів + правила
    percentiles.json      # OSF percentile table (стор. 11)
    templates.json        # шаблони звіту + застереження
  docs/
    SOURCE.md             # звірка контенту з OSF сторінками
    test-engine.js        # тести алгоритму (звірка з OSF Sample Report)
    sample_responses.json # тестові відповіді для ?debug=1
```
